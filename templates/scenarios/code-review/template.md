# 代码审查 PDCA 模板

> **模板类型**: 场景专用模板  
> **适用场景**: 代码审查、Code Review、质量保证活动  
> **输出物类型**: 审查报告 + 改进建议 + 更新后的代码规范  
> **PDCA层级**: 质量控制层（可嵌套在功能开发或迭代的大PDCA中）

---

## 📋 使用说明

本模板专为代码审查场景设计，指导您使用PDCA循环系统化地执行代码审查，提升代码质量。

### 何时使用

- 功能开发完成后，合并到主分支前
- 定期代码质量检查
- 重构后的代码验证
- 新人代码指导和培训
- 技术债务识别和评估

### 审查类型

- [ ] **前置审查**：代码合并前的强制审查
- [ ] **定期审查**：每周/每迭代的例行审查
- [ ] **专项审查**：针对特定模块或问题的深入审查
- [ ] **随机抽查**：随机的代码质量检查
