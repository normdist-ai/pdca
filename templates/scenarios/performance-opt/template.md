# 性能优化 PDCA 模板

> **模板类型**: 场景专用模板  
> **适用场景**: 系统性能调优、瓶颈分析、性能改进  
> **输出物类型**: 性能分析报告 + 优化后的代码/配置 + 性能基准数据  
> **PDCA层级**: 优化层（可嵌套在功能开发或系统维护的大PDCA中）

---

## 📋 使用说明

本模板专为性能优化场景设计，指导您使用PDCA循环系统化地进行性能分析和优化。

### 何时使用

- 系统响应慢，用户投诉
- 性能测试发现瓶颈
- 新功能上线后性能下降
- 定期性能健康检查
- 容量规划前的性能评估

### 优化类型

- [ ] **响应时间优化**: 降低API或页面响应时间
- [ ] **吞吐量优化**: 提高系统并发处理能力
- [ ] **资源利用率优化**: 降低CPU、内存、磁盘IO
- [ ] **数据库优化**: 查询优化、索引优化
- [ ] **前端性能优化**: 加载速度、渲染性能
- [ ] **启动时间优化**: 应用冷启动/热启动时间

cache = redis.Redis(host='localhost', port=6379, db=0)

def get_user_data(user_id):
    cache_key = f"user:{user_id}"
    
    # 尝试从缓存获取
    cached_data = cache.get(cache_key)
    if cached_data:
        return json.loads(cached_data)
    
    # 缓存未命中，从数据库查询
    user = User.query.get(user_id)
    if user:
        # 写入缓存，过期时间300秒
        cache.setex(cache_key, 300, json.dumps(user.to_dict()))
        return user.to_dict()
    
    return None
